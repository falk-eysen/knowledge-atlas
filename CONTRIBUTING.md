# Contributing

Contributions are welcome if they improve the catalog.

## What good contributions do

Prefer contributions that add:

- durable concepts
- original sources
- clear references
- useful corrections
- better maps
- better explanations
- public-safe examples
- explicit uncertainty
- citations

## What to avoid

Avoid:

- random link dumps
- hype without explanation
- private information
- workplace-sensitive details
- copied copyrighted text
- unclear summaries
- uncited factual claims
- personal attacks
- low-quality generated filler

## Public safety

Everything in this repo is public.

Before contributing, check:

```text
Would this be okay for strangers, employers, coworkers, collaborators, and future readers to see?
```

If not, do not add it.

## Preferred structure

Raw public source notes go under `_raw/`.

Synthesized public knowledge goes under `_wiki/`.

Reusable public deliverables go under `_outputs/`.

Schema and workflow material goes under `_system/`.

## OKF compatibility

Normal Markdown files should include YAML frontmatter with a non-empty `type`.

Reserved files:

```text
index.md
log.md
```

Use standard Markdown links.
